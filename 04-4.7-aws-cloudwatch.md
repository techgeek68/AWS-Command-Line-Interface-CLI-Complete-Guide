# AWS CLI — CloudWatch & Logs Operations Guide

## Metrics

### List namespaces
```bash
aws cloudwatch list-metrics --query 'Metrics[].Namespace' --output text | sort -u
```

### List EC2 CPU metrics
```bash
aws cloudwatch list-metrics --namespace AWS/EC2 --metric-name CPUUtilization
```

### Get metric statistics
```bash
aws cloudwatch get-metric-statistics --namespace AWS/EC2 --metric-name CPUUtilization --dimensions Name=InstanceId,Value=<INSTANCE_ID> --start-time $(date -u -d '30 minutes ago' +%FT%TZ) --end-time $(date -u +%FT%TZ) --period 60 --statistics Average Maximum # user input here (instance id)
```

### Latest Lambda datapoint
```bash
aws cloudwatch get-metric-statistics --namespace AWS/Lambda --metric-name Invocations --dimensions Name=FunctionName,Value=<FUNCTION_NAME> --start-time $(date -u -d '15 minutes ago' +%FT%TZ) --end-time $(date -u +%FT%TZ) --period 60 --statistics Sum --query 'Datapoints|sort_by(@,&Timestamp)[-1]' # user input here (function name)
```

## Custom Metrics

### Publish custom metrics
```bash
aws cloudwatch put-metric-data --namespace "Custom/App" --metric-data '[{"MetricName":"RequestCount","Value":1,"Unit":"Count"},{"MetricName":"LatencyMs","Value":132.4,"Unit":"Milliseconds"}]'
```

### Publish with dimension
```bash
aws cloudwatch put-metric-data --namespace "Custom/App" --metric-data '[{"MetricName":"LatencyMs","Dimensions":[{"Name":"Service","Value":"<SERVICE_DIMENSION>"}],"Value":231.2}]' # user input here (service dimension)
```

## Alarms

### Create alarm
```bash
aws cloudwatch put-metric-alarm --alarm-name <ALARM_NAME> --namespace AWS/EC2 --metric-name CPUUtilization --dimensions Name=InstanceId,Value=<INSTANCE_ID> --comparison-operator GreaterThanThreshold --threshold 80 --evaluation-periods 2 --datapoints-to-alarm 2 --period 60 --statistic Average --treat-missing-data notBreaching # user input here (alarm name / instance id)
```

### Describe alarm
```bash
aws cloudwatch describe-alarms --alarm-names <ALARM_NAME> # user input here (alarm name)
```

### Delete alarm
```bash
aws cloudwatch delete-alarms --alarm-names <ALARM_NAME> # user input here (alarm name)
```

### Set alarm state (test)
```bash
aws cloudwatch set-alarm-state --alarm-name <ALARM_NAME> --state-value ALARM --state-reason "Testing alarm" # user input here (alarm name)
```

## Logs

### List log groups
```bash
aws logs describe-log-groups --query 'logGroups[].logGroupName'
```

### Create log group
```bash
aws logs create-log-group --log-group-name <LOG_GROUP_NAME> # user input here (log group)
```

### Set retention
```bash
aws logs put-retention-policy --log-group-name <LOG_GROUP_NAME> --retention-in-days 14 # user input here (log group)
```

### Delete log group
```bash
aws logs delete-log-group --log-group-name <LOG_GROUP_NAME> # user input here (log group)
```

### Create log stream
```bash
aws logs create-log-stream --log-group-name <LOG_GROUP_NAME> --log-stream-name <LOG_STREAM_NAME> # user input here (group / stream)
```

## Ingestion & Tail

### Put log event
```bash
aws logs put-log-events --log-group-name <LOG_GROUP_NAME> --log-stream-name <LOG_STREAM_NAME> --log-events '[{"timestamp":'$(date +%s000)',"message":"Startup complete"}]' # user input here (group / stream)
```

### Get sequence token
```bash
aws logs describe-log-streams --log-group-name <LOG_GROUP_NAME> --log-stream-name-prefix <LOG_STREAM_NAME> --query 'logStreams[0].uploadSequenceToken' --output text # user input here (group / stream)
```

### Tail logs
```bash
aws logs tail <LOG_GROUP_NAME> --since 10m --follow # user input here (log group)
```

### Filter pattern
```bash
aws logs filter-log-events --log-group-name <LOG_GROUP_NAME> --filter-pattern "ERROR" --limit 20 # user input here (log group)
```

## Insights Queries

### Start query
```bash
QUERY_ID=$(aws logs start-query --log-group-name <LOG_GROUP_NAME> --start-time $(date -d '15 minutes ago' +%s) --end-time $(date +%s) --query-string 'fields @timestamp, @message | sort @timestamp desc | limit 20' --query 'queryId' --output text) # user input here (log group)
```

### Get query results
```bash
aws logs get-query-results --query-id <QUERY_ID> # user input here (query id)
```
