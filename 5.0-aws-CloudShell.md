## 1. Overview
AWS CloudShell is a browser-based, pre-authenticated shell (Amazon Linux variant) inside the AWS Console:
- Pre-installed: AWS CLI v2, SDKs (Python, Node), git, jq, curl, zip, Python3, pip, tar.
- Persistent 1 GB home directory per region at `/home/cloudshell-user/`.
- Uses your logged-in console credentials (no local credential file needed).
- Ideal for quick experimentation, automation, troubleshooting, and demos.

## 3. Launching & Session Lifecycle
CloudShell sessions tie to your console login & auto-timeout after inactivity.

### Syntax
- Launch via AWS Console (CloudShell icon).
- Resume: Re-open CloudShell (persistent home restored).
- Timeouts: Session ends → processes killed, ephemeral OS-level installs lost (except `$HOME` content).

### Example
1. Open Console → Click CloudShell icon → Shell prompt appears.
2. Run:
```bash
aws sts get-caller-identity
```
3. Idle > (timeout) → Reopen → Files still at `$HOME`.

## 4. Authentication & IAM Context

### Get current identity
```bash
aws sts get-caller-identity
```

### Assume a role (cross-account)
Syntax:
```bash
aws sts assume-role --role-arn <ROLE_ARN> --role-session-name <SESSION>
```
Example:
```bash
aws sts assume-role \
  --role-arn arn:aws:iam::222233334444:role/CrossAccountRead \
  --role-session-name cloudShellXAcct \
  > /tmp/assume.json

export AWS_ACCESS_KEY_ID=$(jq -r '.Credentials.AccessKeyId' /tmp/assume.json)
export AWS_SECRET_ACCESS_KEY=$(jq -r '.Credentials.SecretAccessKey' /tmp/assume.json)
export AWS_SESSION_TOKEN=$(jq -r '.Credentials.SessionToken' /tmp/assume.json)
```

### Return to original console credentials
```bash
unset AWS_ACCESS_KEY_ID AWS_SECRET_ACCESS_KEY AWS_SESSION_TOKEN
```

## 5. Regions & Multi-Region Workflow

### Show all available AWS regions (EC2)
```bash
aws ec2 describe-regions --query 'Regions[].RegionName' --output text | tr '\t' '\n'
```

### Override region for a single command
```bash
aws s3 ls --region eu-west-1
```

### Iterate across regions
```bash
for R in us-east-1 eu-west-1 ap-southeast-1; do
  echo "Region: $R"
  aws ec2 describe-availability-zones --region "$R" --query 'AvailabilityZones[].ZoneName'
done
```

## 6. Persistent Storage & File Management
- 1 GB in `$HOME` per region.
- Use S3 for larger / archival storage.

### Create script
Syntax:
```bash
echo '<command>' > ~/script.sh
chmod +x ~/script.sh
```
Example:
```bash
echo 'aws ec2 describe-instances --query "Reservations[].Instances[].InstanceId"' > ~/list-ec2.sh
chmod +x ~/list-ec2.sh
./list-ec2.sh
```

### List large files
```bash
find ~ -type f -printf '%s %p\n' | sort -nr | head -10
```

### Archive & upload
```bash
tar czf myhome-backup.tgz ~/*
aws s3 cp myhome-backup.tgz s3://my-backups-bucket/cloudshell/
```

## 7. Package & Dependency Management

### Install OS packages (ephemeral)
```bash
sudo yum install -y tree
tree ~
```
(Note: Some packages may not persist across environment resets.)

### Install Python packages (per-user, persists in `$HOME/.local`)
```bash
pip3 install --user boto3 rich
python3 -c "import rich; print(rich.__version__)"
```

### Virtual environment
```bash
python3 -m venv ~/venvs/tools
source ~/venvs/tools/bin/activate
pip install requests
deactivate
```

### Node (if pre-installed, else nvm script)
```bash
node --version
npm --version
```

## 8. Environment Customization

### Persistent variables (.bashrc)
```bash
echo 'export AWS_DEFAULT_REGION=us-east-1' >> ~/.bashrc
echo 'alias ll="ls -alF"' >> ~/.bashrc
source ~/.bashrc
```

### AWS CLI profiles (for role chaining)
`~/.aws/config` example:
```ini
[profile base]
region = us-east-1

[profile prod-admin]
role_arn = arn:aws:iam::999988887777:role/Admin
source_profile = base
region = us-east-1
```

Use:
```bash
aws sts get-caller-identity --profile prod-admin
```

### Custom prompt segment
```bash
echo 'export PS1="[\u@\h \W $(aws configure get region 2>/dev/null)]$ "' >> ~/.bashrc
source ~/.bashrc
```

## 9. Working with AWS CLI (Syntax + Examples)

### Generic AWS CLI syntax
Syntax:
```bash
aws <service> <operation> [--parameters] [--query '<JMESPath>'] [--output <format>]
```

### Examples
```bash
aws ec2 describe-instances --query 'Reservations[].Instances[].InstanceId'
aws s3 ls
aws iam list-users --output table
aws lambda list-functions --region $REGION
```

### Using query & output
```bash
aws ec2 describe-instances \
  --query 'Reservations[].Instances[].{Id:InstanceId,Type:InstanceType,State:State.Name}' \
  --output table
```

## 10. Scripting & Automation Patterns

### Loop through S3 buckets listing object counts
```bash
for B in $(aws s3api list-buckets --query 'Buckets[].Name' --output text); do
  COUNT=$(aws s3 ls s3://$B --recursive | wc -l)
  echo "$B: $COUNT"
done
```

### Batch tagging EC2 instances
```bash
IDS=$(aws ec2 describe-instances --filters "Name=tag:Env,Values=dev" \
  --query 'Reservations[].Instances[].InstanceId' --output text)
for I in $IDS; do
  aws ec2 create-tags --resources $I --tags Key=Backup,Value=Yes
done
```

### Parallel (xargs) example (careful with rate limits)
```bash
aws ec2 describe-instances --query 'Reservations[].Instances[].InstanceId' --output text | \
tr '\t' '\n' | xargs -n1 -P3 -I{} aws ec2 describe-instance-status --instance-ids {}
```

## 11. SDK (Python boto3) Usage

### Quick inline
```bash
python3 -c "import boto3, json; print(json.dumps(boto3.client('sts').get_caller_identity(), indent=2))"
```

### Script file
`~/list_tables.py`:
```python
import boto3, json
d = boto3.client('dynamodb')
print(json.dumps(d.list_tables(), indent=2))
```
Run:
```bash
python3 ~/list_tables.py
```

### Pagination example
```python
import boto3
s3 = boto3.client('s3')
p = s3.get_paginator('list_objects_v2')
for page in p.paginate(Bucket='my-bucket', Prefix='logs/'):
    for obj in page.get('Contents', []):
        print(obj['Key'])
```

## 15. Security Practices & Least Privilege
- CloudShell runs under your current **IAM identity** → apply least privilege policies.
- Never store raw secrets in plain text: use AWS Secrets Manager or Parameter Store.
- Clear sensitive environment variables after use:
```bash
unset DB_PASSWORD
```
- Limit outbound actions; verify region & account before destructive commands:
```bash
aws sts get-caller-identity
echo "Region: $(aws configure get region)"
```

## 16. Performance & Limits
| Aspect | Typical Behavior / Limit |
|--------|--------------------------|
| Persistent storage | 1 GB per region (`$HOME`) |
| Session timeout | Inactivity triggers termination |
| CPU/RAM | Enough for scripting, not heavy compilation |
| Package persistence | Under `$HOME` (pip --user) persists; system-level might reset |
| Network | Outbound allowed; inbound restricted |
| Long-running tasks | Terminated if session closes |

## 17. Troubleshooting
| Issue | Symptom | Likely Cause | Resolution |
|-------|---------|--------------|-----------|
| Command AccessDenied | AWS CLI errors | IAM policy insufficient | Add necessary permissions / assume proper role |
| Disk full | Write failures | 1 GB quota reached | Clean up old logs / tar & offload to S3 |
| Lost installed yum packages | Missing after restart | Non-persistent system path | Reinstall each session or use script bootstrap |
| Python module not found | ImportError | Installed without --user | Reinstall with `pip3 install --user <pkg>` |
| Region mismatch | Resources not found | Wrong default region | `aws configure set region <region>` or use `--region` |
| STS assume-role fails | AccessDenied | Missing sts:AssumeRole on target role | Update trust / permission policies |
| Terminal frozen | Unresponsive prompt | Network / browser tab stall | Reopen CloudShell; session restarts |
| Timeout of scripts | Session ended | Inactivity threshold reached | Use `screen`/`tmux` (if available) or keep activity—avoid extremely long tasks |
| Git push auth fail | 403 or auth error | Missing token / HTTPS credential | Configure credentials helper / create PAT (if needed) |

## 18. Cleanup & Storage Reclamation

### Find large dirs
```bash
du -sh ~/* | sort -hr | head
```

### Remove caches
```bash
rm -rf ~/.cache/pip/*
```

### Archive & remove
```bash
tar czf logs-archive.tgz ~/logs && rm -rf ~/logs/*
aws s3 cp logs-archive.tgz s3://my-bucket/cloudshell-archives/
```

### Delete old scripts
```bash
find ~/ -name "*.sh" -mtime +30 -delete
```

## 19. Best Practices
- Always `aws sts get-caller-identity` before impactful operations.
- Keep the environment bootstrap script (`~/bootstrap.sh`) to restore packages quickly.
- Use parameterized scripts; avoid hard-coded ARNs/account IDs.
- Store large or binary assets in S3; reserve `$HOME` for scripts + configs.
- Use JMESPath queries to minimize output noise.
- Employ version control (git) for complex automation scripts.
- Clean up roles/temp credentials set via environment variables.
- Tag created resources for traceability (`Env=dev`, `Owner=YourName`).
